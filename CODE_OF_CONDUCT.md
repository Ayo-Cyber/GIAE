# Code of Conduct

GIAE is a community of bioinformaticians, software engineers, and
researchers building open tools for genome interpretation. We want
working on GIAE — opening an issue, reviewing a PR, attending a
discussion — to feel welcoming and productive for everyone.

## Our standards

We expect everyone in the community to:

- **Be respectful** in code reviews, issues, and discussions. Critique
  the work, not the person.
- **Assume good intent.** Most disagreements are misunderstandings, not
  bad faith.
- **Be patient with new contributors.** Everyone was new once. If you
  can answer a "this is probably obvious but…" question with grace,
  please do.
- **Take responsibility.** If you make a mistake, own it and learn.
- **Stay on topic.** This is a science / engineering project — keep
  discussions focused on the work.

We will not tolerate discriminatory or harassing conduct, personal
attacks, or any behaviour that creates an unwelcoming environment.

## Scope

This Code of Conduct applies to all project spaces — GitHub issues and
PRs, discussions, the documentation site, and any official
communication channels — and to anyone representing the project in
public.

## Reporting

If you experience or witness behaviour that violates this Code of
Conduct, please report it privately to the maintainers at
**conduct@giae.dev** (or the email listed in [`pyproject.toml`](pyproject.toml)
if that address isn't yet active).

Reports are kept confidential. We commit to:

1. Acknowledge receipt within **48 hours**.
2. Investigate the report fairly and promptly.
3. Communicate the outcome to the reporter.

## Enforcement

Maintainers will take any action they deem appropriate, including:

- A private reminder of expected standards
- A public clarification of community standards
- A temporary ban from project spaces
- A permanent ban for repeat or severe violations

We aim for proportionate responses. The goal is restoring a healthy
community, not punishment.
